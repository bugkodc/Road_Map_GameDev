# 🗑️ Dispensables

![Dispensables](../../../images/refactoring/dispensables.png)

## What are Dispensables?

**Dispensables** are a group of code smells made up of useless elements that add no real value to the system. Removing them makes the code cleaner, leaner, easier to understand, and far easier to maintain.

> [!TIP]
> Avoid letting garbage pile up in your codebase. The more code there is, the more room there is for bugs. Cleaning up Dispensables reduces the cognitive load on other developers when they read and try to understand the project.

## 📋 List of Code Smells

| # | Code Smell | Short description |
|:-:|-----------|-------------|
| 1 | [Comments](./01-comments.md) | Excessive comments, or comments used to hide dirty code. |
| 2 | [Duplicate Code](./02-duplicate-code.md) | Identical code appearing in multiple places. |
| 3 | [Lazy Class](./03-lazy-class.md) | A class that is too simple and does nothing truly meaningful. |
| 4 | [Data Class](./04-data-class.md) | A class that holds only data (fields, properties, getters/setters) with no behavior. |
| 5 | [Dead Code](./05-dead-code.md) | Code that is never called or used anymore. |
| 6 | [Speculative Generality](./06-speculative-generality.md) | "Just-in-case" code designed too abstractly for situations that have not yet occurred. |

## 🎮 In Game Dev

Dispensables commonly show up in game dev through real-world situations like:
- **Comments**: Long-winded comments explaining the flow of an `Update()` method instead of splitting it into small, self-explanatory methods.
- **Duplicate Code**: Logic for calculating the distance between objects or computing damage copy-pasted across the `Player` and `Enemy` classes.
- **Lazy Class**: Wrapper classes created for Unity components that add no functionality beyond calling the original API.
- **Data Class**: An `ItemData` class that holds only attributes and contains none of the logic for handling that item (for example, consuming itself).
- **Dead Code**: Empty `Start()` or `Update()` methods auto-generated by Unity that the developer forgot to delete, or old debug code that is no longer used.
- **Speculative Generality**: Designing an overly complex Quest system that supports thousands of different quest types, when your game is just a simple shooter that needs only two.
